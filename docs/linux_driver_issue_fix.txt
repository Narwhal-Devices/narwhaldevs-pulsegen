On Ubuntu 22.04, probing for serial devices hangs due to the device's JTAG interface being incorrectly exposed as a serial port. Calling close() on this serial port after sending two bytes causes a 30-second hang. This is because the USB side of the driver is blocking until the data is sent, but it is never sent so this blocks until a 30-second timeout.

To reprocuce:

With nothing else connected, there are two USB serial ports in /dev. The first one is actually a JTAG interface, and misbehaves if treated as a serial port:

$ ls /dev/ttyUSB*
/dev/ttyUSB1 /dev/ttyUSB1
Then in Python:

import serial
ser = serial.Serial('/dev/ttyUSB0') # the JTAG interface
ser.write(b'\x00\x00')
ser.close() # this hangs for 30 seconds

On systems where it may not be ideal to have a misbehaving serial port, e.g. if other software also has some serial port scanning where it sends data to all serial ports and would be vulnerable to hanging, one can add a udev rule to unbind the JTAG interface so that no software sees it at all:

/etc/udev/rules.d/99-alchitry.rules:

# Alchitry Au: interface 0 is JTAG, not a real UART - unbind ftdi_sio
ACTION=="bind", SUBSYSTEM=="usb", DRIVER=="ftdi_sio", ATTRS{idVendor}=="0403", ATTRS{idProduct}=="6010", ATTR{interface}=="Alchitry Au", ATTR{bInterfaceNumber}=="00", RUN+="/bin/sh -c 'echo %k > /sys/bus/usb/drivers/ftdi_sio/unbind'"
